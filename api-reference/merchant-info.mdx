## Visão Geral

Retorna informações completas da conta merchant autenticada, incluindo dados da empresa, configurações de pagamento, limites operacionais e status da conta. Este endpoint é útil para:

- **Dashboard**: Exibir informações da conta
- **Configurações**: Validar limites e recursos disponíveis
- **Integração**: Verificar status e capacidades da conta
- **Compliance**: Dados para relatórios e auditorias

<Info>
**Dados Sensíveis**: Informações de conta são retornadas apenas para a chave API correspondente. Não é possível consultar dados de outros merchants.
</Info>

## Parâmetros da Requisição

### Headers Obrigatórios

<ParamField header="Authorization" type="string" required>
  Chave API no formato Bearer token
  ```
  Bearer pk_sandbox_sua_chave
  ```
</ParamField>

### Query Parameters

<ParamField query="include" type="string">
  Seções adicionais para incluir na resposta (separadas por vírgula):
  - `limits` - Limites operacionais detalhados
  - `settings` - Configurações avançadas
  - `stats` - Estatísticas de uso do mês atual
  - `banking` - Dados bancários para liquidação
</ParamField>

## Exemplo de Requisição

<RequestExample>

```bash cURL
# Consulta básica
curl -X GET https://api.debita.ai/v1/merchant \
  -H "Authorization: Bearer pk_live_sua_chave"

# Consulta completa
curl -X GET "https://api.debita.ai/v1/merchant?include=limits,settings,stats,banking" \
  -H "Authorization: Bearer pk_live_sua_chave"
```

```javascript Node.js
// Consulta básica
const response = await fetch('https://api.debita.ai/v1/merchant', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer pk_live_sua_chave'
  }
});

const merchant = await response.json();
console.log('Empresa:', merchant.business_name);
console.log('Status:', merchant.status);

// Consulta com dados completos
const detailedResponse = await fetch(
  'https://api.debita.ai/v1/merchant?include=limits,stats',
  {
    method: 'GET',
    headers: {
      'Authorization': 'Bearer pk_live_sua_chave'
    }
  }
);

const detailedMerchant = await detailedResponse.json();
console.log('Limites:', detailedMerchant.limits);
console.log('Estatísticas:', detailedMerchant.stats);
```

```python Python
import requests

# Consulta básica
url = 'https://api.debita.ai/v1/merchant'
headers = {
    'Authorization': 'Bearer pk_live_sua_chave'
}

response = requests.get(url, headers=headers)
merchant = response.json()

print(f'Empresa: {merchant["business_name"]}')
print(f'Status: {merchant["status"]}')

# Consulta com dados completos
params = {'include': 'limits,settings,stats,banking'}
detailed_response = requests.get(url, headers=headers, params=params)
detailed_merchant = detailed_response.json()

print(f'Volume mensal: R$ {detailed_merchant["stats"]["monthly_volume"] / 100:.2f}')
```

</RequestExample>

## Campos da Resposta

### Campos Básicos

<ResponseField name="id" type="string">
  ID único do merchant no formato `mer_` seguido de caracteres alfanuméricos
</ResponseField>

<ResponseField name="status" type="string">
  Status atual da conta:
  - `active`: Conta ativa e operacional
  - `inactive`: Conta temporariamente desativada
  - `under_review`: Em análise de compliance
  - `restricted`: Com restrições operacionais
  - `suspended`: Suspensa por violação de termos
</ResponseField>

<ResponseField name="business_type" type="string">
  Tipo de negócio:
  - `individual`: Pessoa física
  - `mei`: Microempreendedor Individual
  - `corporation`: Pessoa jurídica
  - `nonprofit`: Organização sem fins lucrativos
  - `startup`: Startup em fase inicial
</ResponseField>

### Verificação KYC

<ResponseField name="verification.kyc_status" type="string">
  Status do processo de verificação:
  - `pending`: Aguardando documentos
  - `pending_review`: Em análise
  - `verified`: Verificado com sucesso
  - `rejected`: Rejeitado (necessita correção)
</ResponseField>

### Limites Operacionais

<ResponseField name="limits.daily_volume" type="object">
  Limite de volume diário
  <Expandable title="Propriedades do daily_volume">
    <ResponseField name="current" type="integer">
      Volume processado hoje (em centavos)
    </ResponseField>
    <ResponseField name="limit" type="integer">
      Limite diário total (em centavos)
    </ResponseField>
    <ResponseField name="percentage_used" type="number">
      Percentual utilizado do limite
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="limits.single_transaction" type="object">
  Limites por transação
  <Expandable title="Propriedades do single_transaction">
    <ResponseField name="min" type="integer">
      Valor mínimo por transação (em centavos)
    </ResponseField>
    <ResponseField name="max" type="integer">
      Valor máximo por transação (em centavos)
    </ResponseField>
  </Expandable>
</ResponseField>

### Configurações

<ResponseField name="settings.webhook_url" type="string">
  URL configurada para receber webhooks
</ResponseField>

<ResponseField name="settings.auto_settlement" type="boolean">
  Se a liquidação automática está habilitada
</ResponseField>

<ResponseField name="settings.settlement_frequency" type="string">
  Frequência de liquidação:
  - `daily`: Diária (D+1)
  - `weekly`: Semanal
  - `biweekly`: Quinzenal
  - `monthly`: Mensal
</ResponseField>

### Estatísticas

<ResponseField name="stats.success_rate" type="number">
  Taxa de sucesso das transações (percentual)
</ResponseField>

<ResponseField name="stats.average_ticket" type="integer">
  Ticket médio em centavos
</ResponseField>

<ResponseField name="stats.top_payment_method" type="string">
  Método de pagamento mais utilizado
</ResponseField>

### Dados Bancários

<ResponseField name="banking.settlement_bank" type="string">
  Código do banco para liquidação
</ResponseField>

<ResponseField name="banking.settlement_account_type" type="string">
  Tipo da conta:
  - `checking`: Conta corrente
  - `savings`: Poupança
</ResponseField>

### Taxas

<ResponseField name="fees" type="object">
  Taxas aplicadas por tipo de transação (valores em centavos)
  <Expandable title="Tipos de taxa">
    <ResponseField name="pix_dynamic" type="integer">
      Taxa por PIX dinâmico
    </ResponseField>
    <ResponseField name="pix_static" type="integer">
      Taxa por PIX estático
    </ResponseField>
    <ResponseField name="boleto" type="integer">
      Taxa por boleto
    </ResponseField>
    <ResponseField name="chargeback_fee" type="integer">
      Taxa de chargeback
    </ResponseField>
  </Expandable>
</ResponseField>

## Status da Conta

### `active`
- Conta totalmente operacional
- Todos os recursos disponíveis
- Limites normais aplicados

### `under_review`
- Análise de documentos em andamento
- Funcionalidades limitadas
- Ambiente de teste disponível

### `restricted`
- Restrições específicas aplicadas
- Alguns recursos podem estar bloqueados
- Contate o suporte para detalhes

### `suspended`
- Conta temporariamente suspensa
- Transações bloqueadas
- Necessário contato com compliance

## Códigos de Erro

<ResponseField name="401" type="error">
  **Unauthorized** - Chave API inválida ou ausente
</ResponseField>

<ResponseField name="403" type="error">
  **Forbidden** - Acesso negado ou conta suspensa
</ResponseField>

## Monitoramento de Limites

Use este endpoint para monitorar limites e alertar usuários:

```javascript
async function checkLimits() {
  const response = await fetch(
    'https://api.debita.ai/v1/merchant?include=limits',
    {
      headers: { 'Authorization': 'Bearer pk_live_sua_chave' }
    }
  );

  const merchant = await response.json();
  const dailyUsage = merchant.limits.daily_volume.percentage_used;

  if (dailyUsage > 80) {
    console.warn(`Atenção: ${dailyUsage}% do limite diário utilizado`);
  }
}
```
